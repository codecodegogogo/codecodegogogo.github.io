17．如果你的umask设置为022，缺省的你创建的文件的权限为：
A. —-w–w-
B. -w–w—-
C. r-xr-x—
D. rw-r–r–
答案：D
umask是一种权限掩码，用于限制新创建的文件和目录的默认权限。它是一个三位八进制数字，表示要从新文件或目录的默认权限中去除的权限位。例如，如果umask设置为022，则新创建的文件将被授予644的权限（即所有者具有读写权限，其他用户只有读权限），新创建的目录将被授予755的权限（即所有用户都有读取和执行权限，但只有所有者可以写入目录）。
例如，要将umask设置为022，可以执行以下命令：
umask 022

22．在应用程序起动时，如何设置进程的优先级？
A. priority
B. nice
C. renice
D. setpri
答案：B
在Linux系统中，可以使用nice和renice命令来设置进程的优先级。
nice命令
nice命令用于启动一个新的进程，并将其优先级降低（或提高）指定的优先级值。nice命令的语法如下：
nice [-n increment] command
其中，increment表示要增加（或减少）的优先级值。默认情况下，increment为10，表示将进程的优先级降低10个值。例如，要将进程的优先级降低为10，可以执行以下命令：
nice -n 10 command
renice命令
renice命令用于修改已经运行的进程的优先级。renice命令的语法如下：
renice [-n increment] pid
其中，pid表示要修改优先级的进程ID。例如，要将进程ID为1234的进程的优先级降低为10，可以执行以下命令：
renice -n 10 1234
注意，只有root用户才能将进程的优先级增加（即将increment值设为负数）。

23．在 bash 中, 在一条命令后加入”1>&2” 意味着：
A. 标准错误输出重定向到标准输入
B. 标准输入重定向到标准错误输出
C. 标准输出重定向到标准错误输出
D. 标准输出重定向到标准输入
答案：C
在bash中，1表示标准输出（stdout），2表示标准错误输出（stderr），1>&2表示将标准输出重定向到标准错误输出，也就是将输出内容发送到标准错误输出流中。
例如，如果你执行以下命令：
ls /path/to/nonexistent/file 1>&2
这条命令将会把错误信息输出到标准错误输出流，而不是标准输出流。如果/path/to/nonexistent/file不存在，那么会输出类似于以下的错误信息到终端：
ls: cannot access ‘/path/to/nonexistent/file’: No such file or directory
这个技巧常用于将标准输出和标准错误输出合并为一个输出流，以便更好地处理程序的输出。
在上述命令中，1>&2表示将标准输出（文件描述符1）重定向到标准错误输出（文件描述符2）。这意味着，如果命令产生了标准输出，那么它将被发送到标准错误输出流中，而不是终端上。
例如，如果执行以下命令：
ls /path/to/nonexistent/file 1>&2
如果/path/to/nonexistent/file不存在，则会输出以下错误信息到终端上：

ls: cannot access ‘/path/to/nonexistent/file’: No such file or directory
这是因为标准输出被重定向到了标准错误输出，而标准错误输出默认会输出到终端上。
